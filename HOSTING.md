# Hosting this site

The markdown files are the **source**. A static HTML version lives in `site/`,
generated by [`build.sh`](./build.sh) (uses pandoc) and served by Vercel.

```sh
# regenerate site/ after editing any .md
bash build.sh

# preview locally
open site/index.html
```

## Deploy to Vercel

Push this repo to GitHub and import it at vercel.com. `vercel.json` already
points the output at `site/` with clean URLs and no build step on Vercel's side
— the committed HTML is served directly. (pandoc runs locally via `build.sh`;
Vercel never needs it.)

> Workflow: edit markdown → `bash build.sh` → commit both the `.md` and the
> regenerated `site/*.html` → push. Vercel redeploys automatically.
